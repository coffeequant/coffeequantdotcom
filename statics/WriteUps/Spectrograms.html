To avoid any confusions I will list out all the basics before moving to spectrograms.

Frequency: Number of times an event repeats per second. In Physics terms we call samples/seconds as Hz. In our dataset we might change this definition slightly coz of reasons mentioned below.

F = 1/T where T = time taken for an event.

One of the common sampling frequency for Audio is at 44,100 Hz or 44,100 samples per second. Consider a continuous signal

1) signal(t) = Sin(2t)+Cos(11t)

Sampling means taking data at discrete points. In case of audio sampling we would have taken 44,100 samples every second. So t = [0........1second] where “….” denotes division into 1/44,100th of a second. In this case 44,100Hz is the sampling frequency.

As per Shannon's/Nqyuist theorem if the maximum frequency observed in a data is omega Hz then data should be at least sampled by taking points 1/(2 omega) seconds apart. So using F = 1/T definition we have sampling frequency > 2 omega

Ignored Points – This is critical

If I take 1000 points and do a fourier transform then it gets a bit confusing. If the sampling frequency of the data was 44,100 Hz then fourier transform gives 1000 points. From above relation “in red” The maximum frequency omega < ½ x sampling frequency. Points above 500 are greater than omega so can be ignored. Why > omega?

Each point in the spectrum corresponds to frequency bad centered on the frequency (i_index / 1000 x 44,100), where i_index goes from 0 to 1000. So at 500 we get 500 / 1000 x 44,100 or 22,050Hz which is equal to half of sampling frequency!!

Also each of the frequency is actually a band with bandwidth of (2/1000) x 22050 = 44Hz. We won't taking about averaging or bucketing/combining bands. Although an avid reader might use logarithmic bucketing.



Windows

In a frequency plot or FFT plot the time information is lost. Spectrograms are a way of preserving some time information by sacrificing some frequency information. The simple way to achieve this is think of a window moving over the time data and taking FFT only for that portion. Size and shape of the window might be tunable parameters.

A spectrogram is able to do just that by using a time window. It divides the time series into blocks and does a frequency plot of those blocks.

Here 'ks' denotes my frequencies. Generally to make the plot pleasant we do fftshift(ks). This just shifts the 0 frequency to the center rather than at the front and swaps the left and right parts. So if my signal is S[1...3600 points] then fft(S) will be represented by frequencies ks. Although I did say half the frequencies are useless I avoid splitting them and just take an absolute value of the amplitudes

Now here comes the Window

My window is just a plain gaussian lens

So if my window function is exp(-b*(x – a)^2).

1) Parameter b = window size

2) Parameter a = window center or translation if a is changed



So I will be sliding the window in a for loop and taking fft's as follows,

for j in slidewidth:

g=exp(-windowsize*(t-slidewidth[j])**2)

Sg = g*S

Sgt = fft(Sg)

specgram.append(transpose(abs(fftshift(Sgt))))
