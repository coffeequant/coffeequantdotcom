Bias and Variance


Simple questions provide an insiinght into the underlying concept. Suprisingly i have found that candidates memorize the model / procedure but fail to explain the concepts.


Bias => How biased is the model

Variance => How much is the variance in predictions


Let’s add some symbols. To keep it simple I have kept the function as 1 dimensional (so a single feature model).


The orignal data satisfies some pure unknown function which can be assumed


Y = F(X)


The Model which a modeler built has the follwing function


Z = P(X)


There will of course be some difference between Y and Z which will depend on the goodness of the model.


In basic modeling you train your model for a subset of data and then test it against remaining data. Lets say I have 100 data points where each data point is (X,Y) pair. If I divide this complete set into 5 parts then each part will have 20 data points.


I can train my model on part 1 and see how well it can predict part 2


(X1 , Y1 ) (X2 , Y2 ) ................(X20 , Y20 )

Above will be my input to the model where it will try to find the parameters to match Y values for corresponding X values. If model is biased then it will be very good in finding Y1 .... Y20 for corresponding X1 to X20. If I shift to a newer input such as X21 then the output will be very different from Y21

How do we express this anomaly in simpler terms?


E[P(À)] – Y(À) – error => Bias


Isin’t it an elegant way of expressing the crude thought? E stands for expectation of the function and it denotes how output of P(X) changes for different sets of training data for a new input À


error => For now can be assumed to be a noise term whose average value is 0


To be more precise bias doesnt consider the error term as it’s average value is 0


So Bias = E[P(À)] – Y(À)


Variance => E[P(À)2] - ( E[P(À)] )2


Bias => On average how far are we from the actual result. Here average means results for different training data sets.

Variance => How much is the error in the results.


For example with one training set the ouput is very good but with newer set the learning is bad. The best analogy is kids puzzle solving. If they learn to solve a jungle picture based puzzle will they repeat the same success on newer vegetable picture puzzle?


Intuitively there is always a tradeoff and you want a model with low bias and low variance.


Low Bias + Low Variance => Holy Grail below!











Models with low variance tend to be less complex with a simple structure. They also tend to be more robust. Models with low bias will generally be more complex. It’s like learning to solve a very complex puzzle to perfection but you might end up learning that only one puzzle.
