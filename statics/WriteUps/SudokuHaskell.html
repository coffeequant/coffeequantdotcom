Above is the sudoku which we need to solve. The input format is very simple and I have dumped this in the file “sudoku.txt”.

s <- readFile "sudoku.txt"

"0 5 0 0 6 0 0 0 1\n0 0 4 8 0 0 0 7 0\n8 0 0 0 0 0 0 5 2\n2 0 0 0 5 7 0 3 0\n0 0 0 0 0 0 0 0 0\n0 3 0 6 9 0 0 0 5\n7 9 0 0 0 0 0 0 8\n0 1 0 0 0 6 5 0 0\n5 0 0 0 3 0 0 6 0\n\n"

Our strategy is to first generate the possible positions and map each position to the actual sudoku given. We can zip the location index with the above input. Lastly we try to solve this in a searching fashion. Here are the steps in easy to read format

1. Generate location map as a 2D (x,y) system.

2. Zip input matrix with location map ==> zip [1,2,3] [(1,1),(1,2),(2,3)] = [(1,(1,1)),(2,(1,2)),(3,(2,3))]

3. Mark each of the locations with array of possible choices and eliminate those which are not possible

e x y = div (x-1) 3==div (y-1) 3

Above function checks weather its on the same row , column. So as we mark each location we check 
using above function which choices are invalid.

4. Iterate all locations using and mark them

Let's start with step 1

loc = [(i,j) | i ← [1..9] , j←[1..9]]
–- Simple list comprehension for generating 2d array of 9x9

[(p,n) | (p,n) ← zip loc $ map read $ lines s >>= words, n>0]
– n>0 is present to ignore all entries which are 0 in the sudoku input. So this makes a 2D array without the useless elements. 

We know how to solve sudoku, we pick one element and choose the possible values it can get. Let's say the
values possible are (const [1..9]). This function just ignores the next argument. 
So If I do (const [1..9]) (2,3) it will give [1..9].

So when we mark a position we use the above const function to make a decision.

mark :: ((Int,Int),Int) -> ((Int,Int) -> [Int])


-> (Int,Int) -> [Int]

So in above function we have the data as this tuple ((positionx, positiony),value). The accumulator is a function which takes a tuple (x,y) and returns array of ints (this is our const array 1..9). The final argument is the return value which is again a similar function. We are actually storing data inside the function.

Now that signature is a bit clearer, let's use type command to make it less lengthy to write.

Type T = (Int,Int) → [Int]

mark::((Int,Int),Int) → T → T

mark (p@(i,j),n) s q@(x,y) =
  if p==q then [n] else
  if x==i || y==j || e x i && e y j then delete n $ s q else s q
  where e a b = div (a-1) 3==div (b-1) 3


The return type of the function is T,but here we have done partial application. In haskell you can have a function call with incomplete arguments and it's an incomplete function. So “mark” is an incomplete function call which accepts q@(x,y). So we are making the function which carries out the checking but we are still waiting for the searching algo to start. The search will pass all possible choices and return correct valuues. Hope this makes sense, coz this is the most complicated part of this function. To understand it more clearly try this.

:t (mark ((1,2),5) (const [1..9]) )

(mark ((1,2),5) (const [1..9]) ) :: T

This makes sure that we are getting the function back. I have done partial application here to verify. At this point I can't print the result coz there's none. Missing part is to convert this array of functions to another array of functions, but this time with single values!

solve :: [T] -> [T]
solve s = foldr search s loc where
    search p xs = [mark (p,n) s | s <- xs, n <- s p]

And here's the missing part, the call to mark again with a tuple of Int's. This solves the sudoku but the result is still encoded inside functions. We need a special function to map these to actual print values. 

Rest of the code is pasted below.


import List

type T = (Int,Int) -> [Int]

main = do
  s <- getContents
  putStr $ unlines $ map disp $ solve [input s]

solve :: [T] -> [T]
solve s = foldr search s loc where
    search p xs = [mark (p,n) s | s <- xs, n <- s p]

mark :: ((Int,Int),Int) -> T -> T
mark (p@(i,j),n) s q@(x,y) =
  if p==q then [n] else
  if x==i || y==j || e x i && e y j then delete n $ s q else s q
  where e a b = div (a-1) 3==div (b-1) 3

disp :: T -> String
disp s  = unlines [unwords [show $ head $ s (i,j) | j <- [1..9]] | i <- [1..9]]

input :: String -> T
input s = foldr mark (const [1..9]) $
  [(p,n) | (p,n) <- zip loc $ map read $ lines s >>= words, n>0]

loc :: [(Int,Int)]
loc = [(i,j) | i <- [1..9], j <- [1..9]]
